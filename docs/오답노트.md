## #001 VKOSPI 수집 실패 (409 StockConflict)

| 항목 | 내용 |
|------|------|
| **현상** | `domestic.vkospi` 가 null, 리포트에 "미국 VIX (참고)" 표시 |
| **원인** | Naver 모바일 API (`m.stock.naver.com/api/index/VKOSPI/basic`) 가 상장지수에 VKOSPI를 포함하지 않아 409 반환. 08:00 KST 장전 시간에는 영구적으로 실패 |
| **해결** | 4단계 폴백: ① Naver sise_index HTML 파싱 → ② Naver mobile API (2회 재시도) → ③ 전일 `data.json` carry-forward → ④ Yahoo ^VIX |
| **예방** | 16:40 KST `supply_snapshot.js` 에서 VKOSPI를 수집해 `supply.json`에 저장 → 익일 STEP 2.5에서 병합 |
| **관련 파일** | `tools/collectors/domestic.js`, `tools/collectors/supply_snapshot.js`, `tools/main.js` (STEP 2.5) |
| **발생일** | 2026-05-09 ~ 2026-05-12 |

| **현상** | 외국인/기관/개인 수급 바가 리포트에 표시되지 않음 |
| **원인** | Naver `polling.finance.naver.com/api/realtime/domestic/index/KOSPI_INVESTOR` 가 장전(08:00 KST)과 장중에는 `datas: []` 반환. 오후 장 마감(16:30) 후에만 정상 데이터 제공 |

| **예방** | 리포트 수집 시점(08:00 KST)에서는 수급을 직접 수집하려 하지 말 것. 항상 전일 `supply.json`에 의존 |
| **관련 파일** | `.github/workflows/supply-collect.yml`, `tools/collectors/supply_snapshot.js`, `tools/main.js` (STEP 2.5) |

## #003 GitHub Actions 크론 대규모 지연

| 항목 | 내용 |
|------|------|
| **현상** | 08:00 KST 발송 예정 리포트가 실제로 17:00 KST 전후에 도착 (약 9시간 지연) |
| **원인** | GitHub Actions 크론은 UTC 기준이며, 서버 부하에 따라 수십 분~수 시간 지연 가능. 무료 플랜에서 특히 두드러짐 |

| **개선 방향** | 크론 시간을 더 여유 있게 설정하거나 (예: 07:00 KST = 22:00 UTC), 별도 외부 스케줄러(cron-job.org 등)에서 `workflow_dispatch` API 호출 고려 |
| **관련 파일** | `.github/workflows/daily-report.yml` |
| **발생일** | 지속적 (2026-05 기준) |

|------|------|
| **현상** | `[tf-analyst] 한경 컨센서스 수집 실패: Request failed with status code 500` |
| **원인** | `consensus.hankyung.com/apps.analysis/analysis.list` 엔드포인트가 간헐적으로 500 반환. 서버 측 문제로 보임 |

| **관련 파일** | `tools/teams/tf_analyst.js` (`fetchHankyungConsensus`) |
| **발생일** | 2026-05-12 ~ (간헐적) |

## #005 HTML 리포트 서식이 레퍼런스와 불일치

|------|------|
| **현상** | 발송된 메일의 HTML이 `templates/market_report_reference` 디자인과 다름. 헤더 레이아웃, 수급 섹션, 애널리스트 섹션 누락 |

| **예방** | HTML 생성 로직은 반드시 `designer.js` 단일 진입점으로만. `preview_send.js`는 오케스트레이션만 담당 |
| **관련 파일** | `tools/preview_send.js`, `tools/desk/designer.js` |
| **발생일** | 2026-05-12 (발견 및 수정) |

| **현상** | VKOSPI·수급이 리포트에 여전히 빈값으로 표시됨. supply_snapshot이 성공했음에도 다음날 리포트에서 데이터를 읽지 못함 |
| **원인** | 크론이 16:40 KST 대신 01:32 KST(다음날 새벽)에 실행되면, `todayStr`이 다음날 날짜가 됨. 예: supply_snapshot이 `outputs/2026-05-13/supply.json`에 저장했지만, 리포트는 `outputs/2026-05-12/supply.json`을 읽음 → 파일 없음 |

| **예방** | supply_snapshot은 "오늘 날짜"가 아닌 "최근 장 마감 영업일" 기준으로 저장해야 함. 크론 실행 시각에 의존하지 말 것 |
| **관련 파일** | `tools/collectors/supply_snapshot.js` (kstHour < 9 분기 추가), `.github/workflows/supply-collect.yml` |

## 빠른 디버깅 체크리스트

리포트가 안 오면 이 순서로 확인:

```
1. GitHub Actions 실행 여부 확인
   → https://github.com/neofil0308-commits/daily-market-report/actions

2. 오늘 sent.flag 존재 여부 확인
   → outputs/YYYY-MM-DD/sent.flag

3. 로컬 수동 실행으로 직접 발송
   → node tools/main.js --now --dry-run
   → node tools/preview_send.js

4. 오류 로그 확인
   → GA Actions 탭 → 실패한 step 클릭
   → 로컬: 콘솔 출력 확인
```

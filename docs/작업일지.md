사주 지시: "1번부터 가자" — 어제 5차/6차 작업 마무리에서 한경 500을 "서버 일시 장애"로 판단했던 항목 재점검.

#### 진단 (재검증)

- 한경 메인 도메인은 정상 200 응답 → 한경 측 장애 아님
- 우리 코드가 호출하는 `https://consensus.hankyung.com/apps.analysis/analysis.list` → **영구 404** (URL 리디자인됨)
- 신규 엔드포인트 `https://consensus.hankyung.com/analysis/list` → **200 (72KB HTML)**
- 추가 발견: 짧은 UA(`Mozilla/5.0`)는 axios에서 500 반환, 풀 Chrome UA + `Accept: text/html` 헤더 필수 (봇 차단 강화)

**결론**: 어제 "한경 측 일시 장애"는 **오진**. 사실은 한경이 URL을 바꿨고 봇 차단을 강화한 것. 오답노트 #035로 별도 기록.

#### 수정 내역

- **`tools/teams/tf_analyst.js`** — `fetchHankyungConsensus()` 전면 개편
  - URL: `/apps.analysis/analysis.list` → `/analysis/list`
  - 날짜 포맷: `YYYYMMDD` → `YYYY-MM-DD`
  - 카테고리 파라미터: `category=CO` → `report_type=CO`
  - 페이지: `pagenum=1` (페이지번호) → `pagenum=50, now_page=1` (의미 분리)
  - 헤더: 풀 Chrome UA + `Accept: text/html` + `Accept-Language: ko-KR` 추가
  - 파서: 신규 HTML 구조 대응 — 회사명·종목코드가 제목 안에 합쳐졌으므로 `^(.+?)\s*\((\d{6})\)\s*(.*)$` 정규식 분리
  - 신규 필드: `ticker`, `target_price`, `rating` 추가 — Gemini가 추측 대신 한경 원본 값 사용 가능
  - 프롬프트도 신규 필드 노출하도록 갱신

- **`tools/orchestrator.js`** — 4순위 최종 안전망 검색 URL 교체
  - `/apps.analysis/analysis.list?search_value=...` → `/analysis/list?search_value=...`
  - 사주가 종목명 클릭 시 한경에서 해당 회사 리포트 검색 결과로 정상 이동
